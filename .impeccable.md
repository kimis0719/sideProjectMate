## Design Context

### Users

- **Primary**: 한국 개발자 (주니어~미드레벨), 사이드 프로젝트 팀원을 찾고 협업하려는 사람
- **Secondary**: 디자이너, 기획자 등 비개발 직군도 참여 가능
- **Context**: 퇴근 후/주말에 사이드 프로젝트를 빠르게 시작하고 싶은 상황. 가볍고 부담 없이 탐색하는 분위기
- **Job to be done**: 같은 문제에 관심 있는 동료를 찾고, AI 칸반 보드로 바로 실행에 옮기기

### Brand Personality

- **3 words**: 가볍고(Light), 친근하고(Approachable), 실행력 있는(Action-oriented)
- **Tone**: 캐주얼하되 신뢰감 있음. 격식 없지만 허술하지 않음
- **Emotion**: "여기서 바로 시작할 수 있겠다"는 실행 자신감
- **NOT**: 기업적, 무거운, 과시적, 장난스러운

### Aesthetic Direction

- **Visual tone**: Clean, dense, functional — 한국 개발자 커뮤니티 느낌
- **Reference**: peerlist.io (정보 밀도, 무드 참고)
- **Anti-references**: 과도한 여백의 에디토리얼 레이아웃, 스크린샷 중심 랜딩, 어두운 기업 느낌
- **Theme**: 라이트 모드 기본. 다크모드 지원하되 라이트 우선 설계

### Color System

| Token | Value | Usage |
|-------|-------|-------|
| Background | `#FAFAF9` | 페이지 배경 (순백 사용 금지) |
| Card surface | `#FFFFFF` | 카드 배경, 1px border `#E5E7EB` |
| Text primary | `#111827` | 제목, 본문 강조 |
| Text muted | `#6B7280` | 보조 텍스트, 설명 |
| Accent/CTA | `#2563EB` | 버튼, 링크 등 인터랙티브 요소 **전용** |
| Accent hover | `#1D4ED8` | CTA hover 상태 |
| Destructive | `#EF4444` | 삭제, 에러 |
| Border | `#E5E7EB` | 카드, 구분선 |

**Rule**: Blue(`#2563EB`)는 인터랙티브 요소에만 사용. 장식적 사용 절대 금지.

### Typography Hierarchy

| Level | Size | Weight | Color |
|-------|------|--------|-------|
| Section title | 18px | 600 | `#111827` |
| Subtitle | 15px | 500 | `#374151` |
| Body | 14px | 400 | `#6B7280` |
| Caption | 12px | 400 | `#9CA3AF` |

**Rule**: 계층은 size + weight + color 조합으로 표현. 사이즈만으로 구분하지 않음.

### Component Style

- **Cards**: 1px border `#E5E7EB`, radius `10px`, hover시 border `#93C5FD`
- **Dividers**: 1px `#E5E7EB` — 미묘하지만 기능적
- **Tags/Chips**: small rounded pill, `#F3F4F6` 배경
- **Buttons**: ghost(border only) 또는 solid blue — 다른 색상 버튼 없음
- **Logo**: 헤더에만 표시, 다른 곳에 반복하지 않음

### Layout Principles

- 여백보다 정보 밀도를 우선
- 프로젝트 목록은 카드뷰 + 리스트뷰(게시판 스타일) 모두 제공
- 리스트뷰를 기본값으로 설정 (한국 UX 관습)
- 모바일에서도 밀도 유지, 단순히 쌓기만 하지 않음

### Anti-patterns (절대 하지 않을 것)

- `#FFFFFF`를 페이지 배경으로 사용
- 과도한 여백 / 에디토리얼 스페이싱
- Blue를 장식적으로 사용
- 스크린샷 중심 섹션
- 텍스트 사이즈만으로 시각 계층 구분
- 다크모드를 기본값으로

### Design Principles

1. **Dense & Scannable**: 정보 밀도를 유지하되 스캔하기 쉬운 구조. 여백은 그룹핑 용도로만.
2. **Quiet Confidence**: 화려하지 않지만 정돈된 느낌. 색상은 절제하고, 구조로 신뢰감을 줌.
3. **Action-First**: 모든 화면에서 "다음에 뭘 하면 되지?"가 즉시 보여야 함. CTA는 명확하고 유일하게.
4. **Familiar Korean UX**: 한국 사용자에게 익숙한 패턴(리스트뷰, 밀도, 게시판 구조) 존중.
5. **Light by Default**: 라이트 모드에서 최고의 경험을 보장. 다크는 대응, 기본이 아님.

### Accessibility

- WCAG AA 준수 (대비비 4.5:1 이상)
- 키보드 네비게이션: focus-visible 링 유지
- Reduced motion: `prefers-reduced-motion` 미디어쿼리 존중
- 스크린리더: aria-label, sr-only 텍스트 유지
