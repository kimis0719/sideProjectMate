1프로젝트 개요 및 포지셔닝
📌 서비스 정의

Side Project Mate는 개발자를 중심으로 한 사이드 프로젝트 팀 매칭 + 내장 협업 도구(칸반/WBS) 플랫폼입니다. 단순 팀원 매칭을 넘어, 찾은 팀원들과 프로젝트를 직접 관리할 수 있는 All-in-One 도구를 지향합니다.

타겟 유저는 사이드 프로젝트를 진행하고 싶지만 혼자이거나 팀이 부족한 국내 개발자·디자이너·기획자이며, GitHub 연동, Solved.ac 연동, Tech Blog RSS 수집 등 개발자 특화 기능을 전면에 내세웁니다.

팀 매칭 칸반 보드 WBS·간트 실시간 협업 개발자 프로필 GitHub 연동 채팅 Solved.ac

4기술 아키텍처 및 확장성 평가
✅ 강점
  • Next.js 14 App Router: 최신 React 서버 컴포넌트 패턴 적용
  • TypeScript 전면 사용: 타입 안전성과 유지보수성 확보
  • Socket.io 실시간 협업: 칸반 보드에서 Presence, Lock, Sync 구현
  • Cloudinary 이미지 관리: CDN 기반, 서버 부담 없음
  • MongoDB Atlas: 클라우드 관리형 DB, 스케일 유연
  • Zustand + Zundo: 경량 상태 관리 + Undo/Redo
  • 코드 컨벤션 문서화: CLAUDE.md로 팀 기준 명확
  • 컴포넌트 분리: 명확한 폴더 구조와 역할 정의
❌ 약점
  • WBS API 응답 형식 불일치: 컨벤션 미준수 버그 3종
  • GanttChart DOM 직접 조작: setInterval 메모리 누수, 라이브러리 내부 의존
  • 테스트 코드 전무: 단위/E2E 테스트 없음 → 리그레션 위험
  • any 타입 사용: session.user as any 등 타입 안전성 훼손
  • React Hook 규칙 위반: mypage 조건문 이후 useState (런타임 에러 가능)
  • WBS 실시간 동기화 없음: 칸반과 달리 Socket 미적용
  • ESLint/빌드 에러 무시: ignoreBuildErrors: true 설정
  • 배치 삭제 쿼리 버그: MongoDB $pull 쿼리 오류
⚡ 성능·확장성 주요 리스크

6신뢰 시스템 및 안전성 평가

팀 매칭 플랫폼에서 신뢰 시스템은 핵심 자산입니다. 낯선 사람과 프로젝트를 함께 하는 만큼, 유저는 상대방을 검증할 수 있는 장치를 요구합니다.

현재 구현된 신뢰 요소

  • ✅ GitHub 활동 지표 연동
  • ✅ Solved.ac 티어 표시
  • ✅ 기술 스택 아이콘
  • ✅ 가용 시간 스케줄

부재한 신뢰 요소

  • ❌ 팀원 리뷰·평점 시스템
  • ❌ 프로젝트 완료 인증·배지
  • ❌ 신원 확인(휴대폰 인증 등)
  • ❌ 스팸·악성 게시물 신고 기능
  • ❌ 관리자 콘텐츠 모더레이션
  • ❌ 개인정보처리방침 / 이용약관 페이지
특히 개인정보처리방침과 이용약관이 없는 상태에서는 개인정보보호법(PIPA) 위반으로 서비스 운영 자체가 법적 리스크에 노출됩니다. 출시 전 반드시 해결해야 할 사항입니다.
7개선 권고 및 로드맵

상업적 출시 가능 수준(B+ 등급)까지 올리기 위한 우선순위별 개선 권고사항입니다.

🔴 P0 — 출시 전 필수 (1~2주)

P0 채팅 기능 완성
UI 프로토타입 → 실제 Socket.io 연동 완성. 팀 매칭 플랫폼에서 커뮤니케이션 수단 없이는 서비스 의미 없음. 이미 ChatRoom, ChatMessage 모델과 서버 이벤트(join-chat-room 등)가 설계되어 있어 연동 비용은 낮음.
P0 개인정보처리방침 · 이용약관 페이지 신설
법적 필수 요건. 현재 더미 데이터(홍길동 등)가 있는 푸터 대신 실제 법적 정보를 등록해야 합니다. 개인정보보호위원회 고시 기준에 맞춰 작성 필요.
P0 WBS 핵심 버그 수정 (3종)
① 배치 삭제 의존관계 쿼리 버그($pull 오류), ② API 응답 형식 불일치, ③ GanttChart setInterval 메모리 누수. 현재 상태는 데이터 손상 가능성이 있어 프로덕션 불가.
P0 관리자 패널 구현
admin-page-plan.md에 상세 계획이 이미 있음. DB 직접 접근 없이 공통 코드·기술 스택 관리, 악성 유저 차단, 콘텐츠 모더레이션이 가능해야 서비스 운영 가능.

🟠 P1 — 출시 직후 (1~4주)

P1 소셜 로그인 추가 (Google / GitHub OAuth)
개발자 타겟 서비스에서 GitHub OAuth는 사실상 필수. 회원가입 마찰을 80% 이상 줄여 전환율을 크게 향상시킵니다. next-auth에 Provider만 추가하면 되어 구현 비용이 낮습니다.
P1 랜딩 페이지 전면 개편
Hero Section(가치 제안), 서비스 통계(프로젝트 수·멤버 수), 핵심 기능 소개 카드 3개 추가. 비로그인 유저에게 서비스 가치를 60초 내에 전달해야 합니다. CTA도 로그인 여부에 따라 분기 처리.
P1 SEO 기반 구축
OG 태그, robots.txt, sitemap.ts, 프로젝트 상세 generateMetadata 적용. 현재는 검색 엔진에 사실상 노출되지 않아 유기적 성장이 불가능합니다.
P1 팀원 리뷰 · 평점 시스템
프로젝트 완료 후 팀원 간 리뷰를 남기는 기능. 이미 Comment, Post 모델이 있어 확장 용이. 신뢰 시스템의 핵심으로 재방문율과 매칭 품질을 동시에 높임.
P1 배포 인프라 업그레이드
Render.com Free → Starter Plan($7/월) 업그레이드 또는 Railway·Fly.io 이전. 슬립 모드 없이 상시 가동이 가능해야 실사용이 가능합니다. Socket.io 사용으로 상시 연결 유지가 필수입니다.

🔵 P2 — 성장 단계 (1~3개월)

P2 수익화 모델 도입 (Freemium)
무료: 프로젝트 2개, 팀원 5명, 기본 칸반. 유료(월 9,900원): 무제한 프로젝트, WBS/간트 고급 기능, 파일 첨부, 우선 매칭 노출. 초기 MAU 1,000명 돌파 후 Freemium 전환이 적합한 타이밍입니다.
P2 추천 알고리즘 및 매칭 점수 도입
유저의 기술 스택, 가용 시간, MBTI 성향을 기반으로 프로젝트·팀원 추천 점수 계산. LinkedIn의 "당신이 알 수도 있는 사람" 방식. 매칭 플랫폼의 핵심 차별화 요소입니다.
P2 커뮤니티 / 게시판 기능 구현
이미 Post, Comment 모델이 있음. Q&A, 프로젝트 후기, 팁 공유 게시판을 추가하면 리텐션을 높이고 SEO 콘텐츠를 동시에 생산할 수 있습니다. 개발자 커뮤니티로의 진화가 핵심 해자(Moat).
P2 모바일 앱 또는 PWA
알림 푸시, 채팅 응답속도가 모바일에서 결정됩니다. Next.js 기반이므로 PWA 변환이 상대적으로 쉬움. 팀 협업 도구는 모바일 접근성이 MAU 유지에 직결됩니다.

🟢 P3 — 확장 단계 (3개월+)

P3 B2B 기업 채용 연계
사이드 프로젝트 우수 팀을 기업 채용으로 연결하는 파이프라인. Wishket의 프리랜서 채용과 유사하지만 팀 단위 매칭이라는 차별점이 있습니다. MAU 5,000명 이상에서 타당성 검토.
P3 AI 기반 프로젝트 어시스턴트
칸반·WBS에 AI 어시스턴트 도입: 일정 자동 생성, 역할 분담 제안, 기술 스택 추천. OpenAI API 연동으로 차별화. LLM 기반 협업 도구는 2025~2026년 가장 빠르게 성장하는 SaaS 카테고리입니다.
P3 해커톤 / 프로젝트 챌린지 기능
플랫폼 내에서 해커톤·사이드 프로젝트 대회를 주최할 수 있는 기능. 기업 스폰서십 + 참가비 수익 모델 가능. 커뮤니티 활성화와 브랜드 인지도를 동시에 높이는 성장 레버.
8최종 종합 평가
C+
MVP 초기 단계 — 잠재력은 충분, 상업화까지 갭 존재
Side Project Mate는 개발자 특화 팀 매칭 + 협업 도구 통합이라는 독특하고 실현 가능한 포지션을 갖추고 있습니다. 기술 스택 선택과 아키텍처 설계는 합리적이며, 핵심 기능(프로젝트 관리, 지원 시스템, 칸반 보드)은 사용 가능한 수준으로 구현되어 있습니다.

그러나 채팅 미완성, 수익 모델 부재, 관리자 패널 없음, SEO 전무, 법적 고지 누락 등 상업적 출시를 위한 핵심 요건들이 충족되지 않은 상태입니다. 현재 버전은 "포트폴리오 수준의 프로토타입"으로 평가하며, P0 개선 사항을 완료한다면 B 등급 (출시 가능 MVP)으로 재평가 가능합니다.
