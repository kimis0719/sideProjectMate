# Side Project Mate (SPM) — 페이지별 기능 정리

> Stitch 사이트 이용을 위한 서비스 기능 목록화 문서

---

## 서비스 개요

- **서비스명:** Side Project Mate (SPM)
- **스택:** Next.js 14 / TypeScript / MongoDB / Zustand / Socket.io / Tailwind CSS
- **목적:** 사이드 프로젝트 팀 매칭 + AI 기반 프로젝트 관리 + 실시간 협업 플랫폼

---

## 1. 홈 / 랜딩 페이지 (`/`)

| 항목 | 내용 |
|------|------|
| **목적** | 비로그인 사용자에게 서비스 소개, 로그인 시 `/projects`로 자동 리다이렉트 |

### 기능
- 서비스 핵심 가치 제안 (Hero Section)
  - AI 기반 실행 계획 자동 생성
  - 도메인 관심사 기반 팀 매칭
  - 드래그앤드롭 칸반 보드로 실시간 협업
- 3단계 사용 흐름 안내 (프로젝트 등록 → AI 계획 생성 → 팀원 매칭)
- 모집 중인 프로젝트 미리보기 (최대 3개)
- CTA 버튼 (시작하기 → 로그인/회원가입)

## 3. 회원가입 페이지 (`/register`)

| 항목 | 내용 |
|------|------|
| **목적** | 신규 사용자 계정 생성 |

### 기능
- **소셜 가입:** GitHub / Google OAuth (기존 이메일 계정과 자동 연결)
- **이메일 가입:**
  - 이메일 (필수, 유효성 검증)
  - 비밀번호 (필수, 최소 8자 + 3종류 조합)
  - 비밀번호 확인
  - 닉네임 (선택, 랜덤 생성 기능)
  - 휴대폰 번호 (선택, 010-xxxx-xxxx 형식)
- **비밀번호 강도 표시:** 위험 / 보통 / 안전 (3단계 프로그레스 바)
- **약관 동의:** 이용약관 + 개인정보처리방침 링크 (새 탭)

## 4. 온보딩 페이지 (`/onboarding`)

| 항목 | 내용 |
|------|------|
| **목적** | 신규 가입 후 프로필 초기 설정 (3단계 위자드) |

### 기능
- **Step 1 — 관심 도메인 선택:** 다중 선택 (최대 5개), CommonCode API에서 동적 로드
- **Step 2 — 협업 스타일:** 슬라이더(1~100)로 협업 성향 표현, 성격 태그 선택
- **Step 3 — 가용 시간 일정:** 드래그로 주 단위 가용 시간 설정 (평일/주말 구분)
- 이전/다음 버튼, 단계별 진행 바
- 완료 후 `/projects`로 이동
- 이미 완료한 사용자는 자동 리다이렉트

## 8. 프로젝트 생성 페이지 (`/projects/new`)

| 항목 | 내용 |
|------|------|
| **목적** | 신규 프로젝트 생성 |

### 기능

**섹션 1 — 핵심 정보:**
- 프로젝트 제목 (필수, 최대 60자)
- 프로젝트 동기 (필수, 최대 500자)

**섹션 2 — 진행 현황:**
- 현재 단계: Idea / Prototype / MVP / Beta / Launched
- 실행 방식: AI_HEAVY / BALANCED / TRADITIONAL
- 주당 예상 참여 시간: 5h / 10h / 15h / 20h / 30h+

**섹션 3 — 매칭 조건:**
- 관심 도메인 (태그 최대 5개)
- 찾는 사람 (태그 최대 5개, 동적 제안)
- 최대 팀원 수 (2~6명)

**섹션 4 — 부가 정보 (선택, 접기 가능):**
- 상세 배경 설명 (AI 지시서 생성 용도)
- 부가 설명
- 기술 스택 (태그 최대 15개)
- 예상 기간: 1/2/3/6개월 또는 미정
- 링크 (GitHub / 배포 URL / Notion)
- 프로젝트 이미지 (Cloudinary, 드래그앤드롭 순서 변경, 다중 업로드)

## 9. 프로젝트 상세 페이지 (`/projects/[pid]`)

| 항목 | 내용 |
|------|------|
| **목적** | 프로젝트 상세 정보 조회 + 지원/참여 관리 |

### 기능
- **프로젝트 정보:** 이미지 슬라이더, 제목, 설명, 상태 뱃지, 단계/방식/시간/기간, 도메인/역할 태그
- **작성자 프로필 카드** (클릭 시 프로필 페이지 이동)
- **현재 멤버 목록** (카드 형식, 각 프로필 미리보기)
- **관련 링크:** GitHub, 배포 URL, Notion
- **좋아요:** 토글 + 카운트 실시간 업데이트

### 상태별 사용자 인터랙션 (6가지 분기)
| 상태 | 가능한 액션 |
|------|-------------|
| Owner (작성자) | 수정 / 삭제 / 멤버 관리 |
| Closed (모집 마감) | 버튼 비활성화 |
| None (지원 전) | 지원하기 → ApplyModal |
| Pending (지원 대기) | 지원 취소 |
| Accepted (수락됨) | 프로젝트 나가기 |
| Rejected (거절됨) | 재지원 |

### 지원하기 (ApplyModal)
- 지원 동기 (필수, 500자 제한)
- 주당 참여 시간 (필수)
- 추가 메시지 (선택)

### 부가 기능
- **1:1 문의하기:** 작성자에게 채팅 (INQUIRY 카테고리)
- **리뷰 기능:** 프로젝트 완료 후 팀원에게 별점 + 코멘트 작성 (ReviewModal)

## 13. 프로젝트 대시보드 (`/dashboard/[pid]`)

| 항목 | 내용 |
|------|------|
| **목적** | 프로젝트별 협업 도구 통합 대시보드 |

### 기능

**레이아웃 (2열):**
- **좌측 (3/4):**
  - ProjectOverview: 프로젝트 상세 설명, 배경 정보

  - 팀 채팅방 입장 버튼 (멤버 2명 이상, TEAM 카테고리)

**리소스 관리 (플로팅 버튼):**
- ResourceModal: 리소스 추가/수정/삭제
- 타입: LINK (URL) / TEXT (텍스트)
- 카테고리 선택
- Socket.io 실시간 동기화 (전체 멤버 반영)

**Socket.io 이벤트:**
- join-project, project-updated, resource-updated

## 15. 채팅 페이지 (`/chat`)

| 항목 | 내용 |
|------|------|
| **목적** | 실시간 메시징 (DM + 팀 채팅) |

### 기능

**레이아웃 (반응형):**
- **좌측 (PC: 320px / 모바일: 전체):** 채팅방 목록
- **우측 (PC: flex-1 / 모바일: 숨김):** 채팅 창

**채팅방 목록 (ChatRoomList):**
- 채팅방 최신순 정렬
- 상대방 아바타 + 닉네임
- 마지막 메시지 미리보기
- 읽지 않은 메시지 카운트
- 마지막 메시지 시간

**채팅 창 (ChatWindow):**
- 메시지 목록 (스크롤 가능)
- 메시지 입력 + 전송
- 채팅방 정보 헤더
- 나가기 버튼

**채팅방 타입:**
| 타입 | 설명 |
|------|------|
| INQUIRY | 프로젝트 작성자에게 1:1 문의 |
| RECRUIT | 지원자와 작성자 인터뷰 |
| TEAM | 프로젝트 멤버 전체 팀 채팅 |

**실시간 (Socket.io):**
- join-user-room, receive_message, message-received, new-room
- 활성 방: unreadCount 미증가 / 비활성 방: unreadCount 증가

- 개인정보 권리 안내, 보호책임자 연락처

---

## 공통 기능 (전체 페이지)

### 헤더 (Header)
- 네비게이션 바 (로고, 프로젝트, 대시보드, 채팅)
- 로그인/로그아웃 토글
- 사용자 메뉴 드롭다운 (프로필, 마이페이지, 로그아웃)
- 알림 아이콘 (미읽음 카운트)
- Socket.io 연결 (join-user-room)

### 테마
- ThemeProvider: 다크모드 / 라이트모드 지원

### 인증 (NextAuth.js)
- Credentials (이메일/비밀번호) + OAuth (GitHub, Google)
- JWT 세션 기반

### 실시간 통신 (Socket.io)
- 프로젝트: join-project, project-updated, resource-updated
- 채팅: receive_message, message-received, new-room
- 사용자: join-user-room

### 이미지 업로드 (Cloudinary)
- 프로필 아바타, 프로젝트 썸네일/갤러리

### 외부 연동
- GitHub: OAuth + GraphQL API (활동 지표)
- Google: OAuth
- Solved.ac: 알고리즘 티어
