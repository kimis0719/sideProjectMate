## 단계별 개선 계획

### 1단계: CI 통과 (즉시)

`src/components/board/HistoryModal.tsx:134`의 Error 2건 수정

```tsx
// Before
"문자열"  ← JSX 내 따옴표

// After
&quot;문자열&quot;  또는  {'"'}문자열{'"'}
```

이것만 수정하면 CI는 즉시 통과됩니다.

---

### 2단계: 쉬운 Warning 정리

자동 수정 가능하거나 단순 교체인 항목들:

**prefer-const (3건)** — `eslint --fix`로 자동 수정 가능

**no-console (19건)** — 디버깅용 `console.log`를 제거하거나 `console.warn`/`console.error`로 변경

**no-img-element (21건)** — `<img>` → Next.js `<Image />` 컴포넌트로 교체
```tsx
// Before
<img src={url} />

// After
import Image from 'next/image';
<Image src={url} width={W} height={H} alt="설명" />
```

---

### 3단계: lint-staged에 ESLint 추가

`.lintstagedrc.js` 수정:

```js
module.exports = {
  '*.{ts,tsx}': ['eslint --fix', 'prettier --write'],
  '*.{js,jsx,mjs,cjs}': ['prettier --write'],
  '*.{json,md,css}': ['prettier --write'],
};
```

이렇게 하면 **수정한 파일만** ESLint를 통과하면 되므로, 기존 290개 `any`를 한 번에 고칠 필요 없이 점진적으로 정리됩니다.

> 주의: 이 변경은 2단계 완료 후 적용해야 합니다.
> 그렇지 않으면 `any`가 포함된 파일을 건드릴 때마다 관련 없는 `any` 수정까지 강제됩니다.
> 필요하다면 lint-staged에서 `--rule '{"@typescript-eslint/no-explicit-any": "off"}'`로 일시 제외 가능합니다.

---

### 4단계: 점진적 any 제거

- 파일을 수정할 때마다 해당 파일의 `any`를 함께 정리
- `any` → `unknown`, 구체적인 타입, 또는 제네릭으로 교체
- 충분히 줄어들면 나머지 일괄 정리 후 `no-explicit-any`를 `"error"`로 격상

---

### 5단계: exhaustive-deps 검토 (12건)

이 규칙은 기계적으로 수정하면 안 됩니다.
- 의도적으로 마운트 시 1회만 실행하는 경우 → `// eslint-disable-next-line` 허용
- 실제 의존성 누락인 경우 → deps 배열에 추가

각 케이스를 개별 확인 후 수정이 필요합니다.
