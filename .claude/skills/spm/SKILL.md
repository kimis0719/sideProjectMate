# SPM 커맨드 안내

> ⚠️ `/spm`은 하위 호환용 진입점입니다.
> 새 커맨드 체계로 이전되었으니 아래 커맨드를 사용하세요.

## 커맨드 체계

| 커맨드                  | 역할                                              |
| ----------------------- | ------------------------------------------------- |
| `/spm-start [작업내용]` | 작업 시작 (워크존 등록 + 컨텍스트 로딩)           |
| `/spm-commit`           | 중간 커밋 (테스트 검증 + AI 커밋 메시지)          |
| `/spm-done`             | 작업 종료 (work-log 생성 + 문서 갱신 + 구역 해제) |
| `/spm-revert`           | 작업 롤백 (work-log 기반 git revert)              |

---

## 하위 호환 처리

인자에 "종료", "끝", "완료", "done", "finish" 등이 포함되어 있으면
`/spm-done` 플로우를 실행합니다.

그 외에는 `/spm-start` 플로우를 실행합니다.
